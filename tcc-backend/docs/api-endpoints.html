Documentacao da API - TCC Backend

Projeto: tcc-backend
Data da geracao: 06/04/2026
Base URL padrao: http://localhost:8080

Resumo de autenticacao
  • Rotas publicas: POST /api/auth/register, POST /api/auth/login, Swagger/OpenAPI.
  • Demais rotas: exigem Authorization: Bearer <jwt>.
  • O endpoint POST /api/auth/logout limpa o contexto atual, mas nao revoga o token JWT emitido.

1. AuthService

Responsavel pelo cadastro publico de aluno, autenticacao via JWT e logout da sessao atual.

EndpointMetodo do ServiceFuncaoAutenticacao
POST /api/auth/registerregister(RegisterRequest)Cria um usuario do tipo ALUNO, cria o registro de aluno vinculado e devolve um JWT.Publico
POST /api/auth/loginlogin(LoginRequest)Autentica email e senha e devolve um JWT valido para as rotas protegidas.Publico
POST /api/auth/logoutlogout()Limpa o contexto de seguranca da requisicao atual e retorna mensagem textual.JWT

Metodos publicos

MetodoFuncao
registerNormaliza nome, email e RA; valida duplicidade de email; persiste usuario e aluno em transacao; gera token.
loginDelegaa autenticacao ao AuthenticationManager, busca o usuario e gera token JWT.
logoutLimpa SecurityContextHolder e devolve confirmacao de logout.

POST /api/projetos/{id}/recrutarrecrutar(projetoId, usuarioId)Aprova ou cria inscricao de um aluno como colaborador do projeto.JWT

Metodos publicos

deleteExclui um projeto se o usuario autenticado for seu dono.
recrutarAdiciona ou aprova um aluno como colaborador e gera notificacao.
